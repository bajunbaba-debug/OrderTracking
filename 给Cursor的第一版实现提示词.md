# 给 Cursor 的第一版实现提示词

你现在接手 `/Users/jhonsimsim/MVP/OrderTracking` 项目。请按产品经理整理稿做一版“网页版 + 本地部署服务器”的订单项目分析系统原型。

第一版不是继续做 Excel 文件，而是做一个能在本机浏览器打开的 Web 系统：

- 第一次数据从 `项目统计2026.xlsm` 导入。
- 导入后数据写入本地数据库。
- 后续新增订单/项目明细，改为在网页界面录入和维护。
- Excel 只作为初始化数据源和后续导出格式，不再作为主数据源。

## 最高优先级要求

1. 不要删除、覆盖、破坏原始文件。
2. 不要直接修改 `项目统计2026.xlsm` 原文件。
3. 不要把第一版做成单纯 Excel 加工脚本。
4. 第一版要能本地启动服务器，并通过浏览器访问。
5. 第一版先做核心闭环：Excel 导入、数据入库、看板展示、网页录入、风险清单。
6. 页面要能给甲方快速看效果，不要只做接口或命令行。
7. 所有口径以 `订单项目Excel需求梳理_PRD_流程图.md` 为准，不能随便删掉 P1/P2 算法，只是第一版不要让算法压垮主看板。

## 核心口径

以当前系统日期为准；如果为了复现 PRD 数据，可以在开发环境提供一个可配置的“统计日期”，默认 `2026-06-18`。

- 总复杂度 = `预计`
- 设计完成为空 = 未完成
- 设计完成非空 = 已完成
- 交期小于统计日期 = 已超期
- 交期在统计日期到统计日期 + 7 天 = 7天内
- 交期在统计日期 + 8 到统计日期 + 14 天 = 8-14天
- 交期在统计日期 + 15 天以后 = 15天以上
- 交期为空 = 交期缺失
- P2 = 预计
- P20 = 3
- P1：若型号能解析出有效数量 x，则 P1 = 预计 / x；否则 P1 = 预计 / 数量
- 风险规则参考 PRD：
  - 未完成且交期已过 = 红色
  - 未完成且 7 天内到期且总复杂度较高 = 红色
  - 未完成且 7 天内到期 = 橙色
  - 未完成且 8-14 天内到期 = 黄色
  - 发布超过 21 天仍未完成 = 黄色
  - 关键字段缺失 = 数据异常
  - P1 明显高于 P10 或 P2 明显高于 P20 = 算法关注

第一版中“总复杂度较高”可暂定为 `>= 3`，并在代码里集中配置，方便后续调整。
